Guia de configuracion ESP32

Materiales necesarios

Para cada canal de disparo necesitas un modulo ESP32 con un relay o MOSFET de potencia. Aqui el listado:

  • ESP32 DevKit v1 (cualquier variante con WiFi)
  • Modulo relay de 5V (recomendado: SRD-05VDC-SL-C) o MOSFET IRF540N para cargas inductivas
  • Fuente de alimentacion 5V/1A por modulo
  • Cable de par trenzado para conectar al iniciador electrico
  • Caja IP65 para uso exterior
⚠ Seguridad: Siempre trabaja con los iniciadores electricos DESCONECTADOS durante la instalacion del hardware. Nunca conectes un iniciador mientras el sistema esta energizado.

Cableado del relay

Conecta el modulo relay al ESP32 segun esta tabla. Usa el GPIO 2 por defecto (configurable en el firmware):

Instalar el firmware

1

Instalar Arduino IDE y la placa ESP32

Descarga Arduino IDE 2.x. En Preferencias agrega: https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json y luego instala la plataforma "ESP32 by Espressif".

2

Abrir el firmware

En PyroShow Pro ve a Herramientas → Firmware ESP32 → Abrir en Arduino. El archivo es esp32_firmware/esp32_canal.ino.

3

Configurar credenciales WiFi e IP

Edita las primeras lineas del archivo con tu SSID, contrasena e IP estatica unica para cada modulo.

4

Flashear al ESP32

Conecta el ESP32 por USB. Selecciona placa "ESP32 Dev Module" y el puerto correcto. Presiona Subir.

Codigo del firmware

// Firmware PyroShow Pro — Canal de Disparo v2.0
// Cambia SSID, PASSWORD e IP_LOCAL para cada ESP32

#include <WiFi.h>
#include <WebServer.h>

const char* SSID     = "TuRedWiFi";
const char* PASSWORD = "TuContrasena";

IPAddress IP_LOCAL(192, 168, 1, 101); // IP unica por modulo
IPAddress GATEWAY (192, 168, 1,   1);
IPAddress SUBNET  (255, 255, 255, 0);

const int FIRE_PIN          = 2;
const int DURACION_PULSO_MS = 500;

WebServer servidor(80);

void setup() {
  Serial.begin(115200);
  pinMode(FIRE_PIN, OUTPUT);
  digitalWrite(FIRE_PIN, LOW);

  WiFi.config(IP_LOCAL, GATEWAY, SUBNET);
  WiFi.begin(SSID, PASSWORD);
  while (WiFi.status() != WL_CONNECTED) { delay(500); }

  servidor.on("/ping", []() {
    servidor.send(200, "text/plain", "OK");
  });

  servidor.on("/fire", []() {
    digitalWrite(FIRE_PIN, HIGH);
    delay(DURACION_PULSO_MS);
    digitalWrite(FIRE_PIN, LOW);
    servidor.send(200, "text/plain", "OK FIRED");
  });

  servidor.on("/status", []() {
    String r = "{\"ip\":\"" + WiFi.localIP().toString() +
               "\",\"rssi\":" + WiFi.RSSI() + "}";
    servidor.send(200, "application/json", r);
  });

  servidor.begin();
}

void loop() {
  servidor.handleClient();
}

Preguntas frecuentes

El ESP32 no responde al ping

Verifica que el ESP32 y la computadora esten en la misma red WiFi. Comprueba que la IP estatica configurada no este siendo usada por otro dispositivo. Abre el monitor serial de Arduino para ver los logs de conexion.

¿Puedo usar mas de 20 ESP32?

La version Enterprise permite extender el limite. Contacta a ventas para configuraciones especiales de mas de 20 canales.

¿Que pasa si la red WiFi falla durante el espectaculo?

Se recomienda usar un router WiFi dedicado solo para el sistema de disparo, sin otros dispositivos conectados. Un router 5GHz de doble banda con los ESP32 en 2.4GHz da la mejor estabilidad.
